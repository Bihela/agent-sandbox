# Agent Instructions for Vibecoding

When working on this project (Agent Sandbox), you must adhere to the following documentation rules:

1. **Architecture Document (`docs/architecture.md`)**:
   - Keep this document updated with an in-detail and granular architecture of the project.
   - Every time a new feature is added, or an architectural change is made, update this document.

2. **Memory Bank Document (`docs/memory_bank.md`)**:
   - Keep a log of decisions made during development, especially things we decided *not* to use or paths we abandoned.
   - Include the reasoning behind these decisions so we don't repeat the same mistakes or explore dead ends again.

**Crucial**: Every time you are given a task that affects architecture or strategic decisions, you must autonomously update these documents accordingly as part of the task execution.
