# Dataset Card: Agent Sandbox Benchmark

## Dataset Overview
This dataset contains high-fidelity negotiation trajectories generated by the **Agent Sandbox**. It is designed for researchers studying multi-agent systems, negotiation strategies, and LLM reasoning depth.

## Data Structure
Each simulation record includes:
- **Trajectory**: Full JSON-formatted conversation history in ACP (Agent Communication Protocol).
- **Metadata**: 
  - `seed`: Integer used for reproducible model generation.
  - `model_name`: The LLM used (e.g., Mistral-7B, Llama-3-8B).
  - `strategies`: Negotiator strategy presets (Aggressive, Balanced, etc.).
- **Metrics**:
  - `reasoning_depth`: Character count of the agent's internal reasoning chain.
  - `decision_latency`: Response time in milliseconds.
  - `token_usage`: Exact token count for the transaction.
  - `outcome`: Categorized as `agreement`, `timeout`, or `rejection`.

## Hardware Metadata
- **Generator Hardware**: Hybrid (Local CPU/GPU + Cloud TPU/GPU Acceleration).
- **Inference Engines**: 
  - Local: Ollama (Mistral-7B, Llama-3-8B).
  - Cloud: Groq (Llama-3.1-8B, Llama-3.3-70B).

## Model Diversity & Research Value
This dataset contains a mix of model architectures:
- **Small Models (7B-8B)**: Provides baseline data for efficient edge-computing agents.
- **Large Models (70B)**: Provides high-fidelity reasoning benchmarks.
All records are tagged with `model_name` for comparative analysis. The transition from local Mistral to cloud Llama 3.3 enhances the dataset's value by providing more complex negotiation trajectories.

## Usage
Datasets are exported in `.json` and `.csv` formats. Use the [Dataset Analytics Tool](metrics/) for automated failure taxonomy and strategy comparison.
